
这篇文章名为 **《Transformer Inference Arithmetic》**，是一篇通过**第一性原理推理**来分析大语言模型（LLM）推理性能的技术文章。以下是核心要点：

---

## 1. KV 缓存
- 自回归推理时，KV 缓存存储每层的 Key/Value 向量，避免重复计算，**以存储换计算**。
- 每 token 存储量 = `2 × 2 × n_layers × n_heads × d_head` 字节。
- 以 A100 为例，一个 52B 模型计算 KV 时，**内存带宽 vs FLOPs 的临界点约为 208 个 token**。
## 详细计算过程

这个 208 的临界点来自 **A100 GPU 的算力与内存带宽之比**。

### 已知参数

| 参数 | 值 | 说明 |
|------|-----|------|
| A100 FP16 算力 (A_f) | **312 × 10¹² FLOPs/s** | 每秒浮点运算次数 |
| A100 内存带宽 (A_bw) | **1.5 × 10¹² bytes/s** | 每秒读写字节数 |

### 核心思路

计算 KV 需要两个步骤：
1. **从内存加载权重**（Wₖ, Wᵥ 等）→ 受内存带宽限制
2. **用权重进行矩阵乘法** → 受 FLOPs 限制

两者是**流水线并行**的（一边加载一边计算），瓶颈取决于谁更慢。

### 计算步骤

**① 计算 KV 的内存加载量（每 token）**

对于 52B 模型（d_model=8192, n_layers=64）：

```
KV 内存加载量 = 2 × 2 × n_layers × d_model²
              = 4 × 64 × 8192²
              = 4 × 64 × 67,108,864
              = 17,179,869,184 bytes
              ≈ 17.2 GB

KV 缓存每 token = 2 × 2 × n_layers × n_heads × d_head
                  = 4 × 64 × 8192    ← 注意这里 n_heads × d_head = d_model = 8192
                  = 2,097,152 bytes
                  ≈ 0.002 GB / token

```


> 第一个 ×2：K 和 V 两个向量  
> 第二个 ×2：每个参数占 2 字节（FP16）  
> n_layers × d_model²：每层 KV 权重的参数量

**② 计算 KV 的 FLOPs（每 token）**

```
KV FLOPs = 2 × 2 × n_layers × d_model²
         = 17,179,869,184 FLOPs
         ≈ 17.2 GFLOPs
```

> 第一个 ×2：K 和 V 两个矩阵乘  
> 第二个 ×2：每个矩阵乘的 multiply-add 操作系数  
> 每层矩阵乘：tₑ × Wₖ（d_model × d_model），FLOPs = 2 × d_model²

### ③ 计算内存加载时间 vs 计算时间（对于 1 token）

```
内存加载时间 = 加载量 ÷ 内存带宽
            = 17.2 GB ÷ 1.5 TB/s
            = 17.2 × 10⁹ ÷ 1.5 × 10¹²
            ≈ 0.0115 秒 = 11.5 毫秒

计算时间     = FLOPs ÷ 算力
            = 17.2 GFLOPs ÷ 312 TFLOPs/s
            = 17.2 × 10⁹ ÷ 312 × 10¹²
            ≈ 0.000055 秒 = 0.055 毫秒
```

**内存加载时间是计算时间的 208 倍！**

```
11.5 ms ÷ 0.055 ms ≈ 208
```

### ④ 临界点的含义

这意味着：

- **加载一次权重到 GPU** 需要 **11.5 毫秒**
- **计算 1 个 token 的 KV** 只需要 **0.055 毫秒**
- 在等待权重加载完成的 **11.5 毫秒内**，理论上可以计算完 `11.5 ÷ 0.055 ≈ 208` 个 token 的 KV

所以：
| 场景 | 瓶颈 | 原因 |
|------|------|------|
| **≤ 208 tokens** | **内存带宽受限** | 权重还没加载完，计算就已经完成了，GPU 计算单元在空等 |
| **> 208 tokens** | **FLOPs 受限** | 计算量太大，计算时间超过了权重加载时间 |

### 简化公式

临界点 token 数 = **算力 ÷ 内存带宽**

```
临界点 = A_f ÷ A_bw
       = 312 × 10¹² ÷ 1.5 × 10¹²
       = 208
```

这是一个**纯硬件参数决定的常数**，与模型结构无关。对于任何运行在 A100 上的模型，只要操作是**每参数 1 次 multiply-add**（即 FLOPs = 参数量 × 2），临界点都是 208。

> 文章中提到："If we used the rest of our weights to do a full forwards pass on our context, it's also 208"——因为完整前向传播的 FLOPs 和内存加载量都同时乘以了 6（QKV + 输出投影 + MLP×2，约 2×12×n_layers×d_model²），所以比值不变。

## 2. 容量
- 52B 模型的权重约 104GB，A100（40GB 显存）放不下，至少需要 3-4 个 GPU。
- 扣除权重后剩余空间用于 KV 缓存，每 token 约 0.002GB，可存约 8000~23000 个 token。

## 3. 模型并行
- 采用**张量并行**将权重切分到多 GPU，在每层注意力与 MLP 末尾各需 2 次通信（all-reduce）。
- 每层通信总量：`4 × (N-1) × d_model / N` 字节，A100 通信带宽 300GB/s。

## 4. 延迟计算
- **小批量（内存带宽受限）**：延迟 ≈ `2 × P × bytes / (N × 内存带宽)`
- **大批量（FLOPs 受限）**：延迟 ≈ `2 × P × B / (N × FLOPs)`
- 260B 模型在 16 个 GPU 上：小批量约 22ms/token，大批量约 53ms/512 tokens。

## 5. 批量大小
- 内存/FLOP 分界线 = `A_bw / A_f = 208`。
- FLOPs 受限时更高效，但 batch size 增大到一定程度后不再加速。
- 每个步骤的 **FLOPs/字节** 应大于硬件比（A100 为 1040）以避免通信拖累。

## 6. FLOPs 计算细节
- 逐层分解：QKV 投影 `2×3×d_model²`、输出投影 `2×d_model²`、前馈网络 `2×8×d_model²`，合计约 `2 × 12 × n_layers × d_model² ≈ 2P`。
- Softmax、LayerNorm 等向量操作的 FLOPs 占比极小（~0.1%）。

## 7. 中间内存成本
- 向量操作（softmax、ReLU、dropout 等）虽 FLOPs 少但**内存带宽受限**，在小模型上占 43% 时间，大模型（52B）上仅约 5%。

## 8. 与真实基准对比
- 使用 NVIDIA FasterTransformer 实测 13B 模型，理论值与实测结果在考虑带宽利用率和中间激活后基本吻合。

### 一句话总结
这篇文章用简洁的数学模型系统性地阐释了 Transformer 推理中的核心性能瓶颈（内存带宽 vs FLOPs）、KV 缓存、模型并行通信成本与延迟估算，是一份非常实用的大模型推理性能参考。



百万级上下文（如 GPT-4 的 128K、Claude 的 200K、Gemini 的 1M、以及一些开源模型的 1M+）之所以成为可能，是因为近年来出现了**多种突破性技术**的组合，它们共同解决了 KV 缓存在超长上下文下的**显存爆炸**和**计算爆炸**问题。

---

## 核心问题：为什么朴素方案做不到百万上下文？

回顾之前的公式，对于 52B 模型：

```
KV 缓存 = 0.002 GB  / token
100 万 token = 0.002 × 1,000,000 = 2000 GB = 2 TB
```

**2TB 显存**——即使 8 张 A100（320GB）也远远不够。而且注意力计算的复杂度是 O(N²)，100 万 token 的计算量更是天文数字。

---

## 实现百万上下文的关键技术

### 1️⃣ 位置编码改进：RoPE + 位置插值

原始 Transformer 用绝对位置编码，模型没见过的位置（> 训练时的上下文长度）表现很差。

**改进方案：**
- **RoPE（旋转位置编码）**：将位置信息编码到 Q 和 K 向量中，具有相对位置衰减特性
- **位置插值（Position Interpolation）**：将长上下文的位置编码"压缩"映射到模型见过的位置范围内
- **NTK-aware 缩放**：调整 RoPE 的旋转频率，保持高频细节的同时扩展低频范围
- **YaRN**：结合插值和 NTK 的混合方法，用更少的微调实现更大的扩展

> 本质：让模型"理解"它从未见过的新位置。

### 2️⃣ 稀疏注意力 / 近似注意力

不再计算**所有 token 对之间**的注意力，而是只计算**一部分**：

| 方法 | 原理 | 代表 |
|------|------|------|
| **滑动窗口注意力** | 每个 token 只看附近 w 个 token | Mistral, Gemma |
| **全局 + 局部注意力** | 每 N 个 token 设一个"锚点"做全局注意力，其余做滑动窗口 | Longformer, BigBird |
| **稀疏注意力** | 按固定模式选择 attention 对（如膨胀窗口、全局、随机） | Sparse Transformers |
| **查询缓存** | 只保留"重要"的 KV 条目，丢弃不重要的 | H₂O, StreamingLLM |

**StreamingLLM（2023）** 发现：即使只保留 **前 4 个 token + 最近 N 个 token** 的 KV 缓存，模型也能稳定生成，这直接让理论上下文无限扩展。

### 3️⃣ KV 缓存压缩与量化

| 技术 | 效果 |
|------|------|
| **KV 缓存量化**（如 KIVI） | 将 KV 从 FP16 降为 INT4/INT8，缓存大小直接缩小 2~4 倍 |
| **KV 缓存剪枝** | 删除对后续生成"不重要"的 token 的 KV 值 |
| **KV 缓存融合** | 合并相似 token 的 KV 值 |

### 4️⃣ 高效的注意力实现：Flash Attention 系列

这是最底层的工程优化，**直接改变注意力计算方式**：

| 版本 | 改进 |
|------|------|
| **FlashAttention** | 将注意力计算分块（tiling）在 SRAM 中完成，避免把 N×N 注意力矩阵写回 HBM，**显存降低 O(N²) → O(N)** |
| **FlashAttention-2** | 进一步减少非矩阵乘操作，提高并行性 |
| **FlashAttention-3** | 利用 Hopper 架构的新特性，达到接近理论极限 |

FlashAttention 让长上下文的注意力计算在**显存和时间上都变得可行**——不再需要显式构造 N×N 的注意力矩阵。

### 5️⃣ Ring Attention / 分布式注意力

当上下文超过单 GPU 显存时，将 KV 缓存**分布在多个 GPU 上**：

```
GPU 0: token 0 ~ 250K    的 KV 缓存
GPU 1: token 250K ~ 500K 的 KV 缓存  
GPU 2: token 500K ~ 750K 的 KV 缓存
GPU 3: token 750K ~ 1M   的 KV 缓存
```

每个 GPU 像接力一样将自己的 KV 块传给下一个 GPU，**环形流水线**式地完成完整注意力计算（Ring Attention, 2023）。

### 6️⃣ 训练时长上下文扩展

即使有上述技术，模型也需要**训练或微调**来适应长上下文：

- **渐进式扩展**：先在 4K 上训练 → 再扩展到 32K → 再到 128K → 再到 1M
- **长上下文数据**：在训练混合中加入长文档、书籍、代码库等数据
- **位置编码微调**：用长数据微调位置编码层，让 RoPE 适应新位置

## 各模型的实际方案

| 模型 | 最大上下文 | 关键技术 |
|------|-----------|---------|
| **Claude 3.5** | 200K | 专有架构 + 高效注意力 |
| **Gemini 1.5 Pro** | 1M~10M | MoE + 稀疏注意力 + 分布式 |
| **GPT-4 Turbo** | 128K | 优化的 FlashAttention + 位置插值 |
| **Mistral** | 32K | 滑动窗口注意力（8K window） |
| **Yi-34B-200K** | 200K | RoPE + 位置插值 + LongCtx |
| **自定义微调模型** | 1M+ | YaRN + FlashAttention + Ring Attention |

## 一句话总结

百万上下文不是某个单一技术的突破，而是**位置编码改进（RoPE/插值）→ 注意力计算优化（FlashAttention）→ KV 缓存压缩（量化/剪枝）→ 分布式计算（Ring Attention）→ 超长训练**这五层技术栈的**系统性工程**。其中 FlashAttention 是最底层的核心突破，它让 O(N²) 的注意力矩阵不再需要全部显式存储，从根本上解除了显存对上下文长度的硬限制。






## LLM（Transformer）架构中的前向传播与反向传播详解

## 一、整体架构概览

一个标准的 Decoder-Only LLM（如 GPT 系列）由以下组件堆叠而成：

```
输入 Token IDs
     │
     ▼
┌──────────────┐
│ Token Embedding │  (词嵌入矩阵 W_e)
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ Positional Encoding │  (RoPE / 可学习位置编码)
└──────┬───────┘
       │
       ▼  ┌─────────────────────────────────┐
       │  │     Transformer Block × N        │
       │  │  ┌───────────────────────────┐   │
       │  │  │ 1. LayerNorm              │   │
       │  │  │ 2. Multi-Head Attention   │   │
       │  │  │    ├─ Q/K/V 线性投影       │   │
       │  │  │    ├─ 缩放点积注意力       │   │
       │  │  │    └─ 输出投影 (W_o)       │   │
       │  │  │ 3. 残差连接 + Dropout     │   │
       │  │  │ 4. LayerNorm              │   │
       │  │  │ 5. FFN (MLP)              │   │
       │  │  │    ├─ W₁: d→4d + 激活函数  │   │
       │  │  │    └─ W₂: 4d→d            │   │
       │  │  │ 6. 残差连接 + Dropout     │   │
       │  │  └───────────────────────────┘   │
       │  └─────────────────────────────────┘
       │        重复 N 层（如 32/64/96 层）
       ▼
┌──────────────┐
│ Final LayerNorm │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ LM Head (W_out) │  (d_model → vocab_size，即 Unembedding)
└──────┬───────┘
       │
       ▼
    logits (每个 token 的预测分数)
       │
       ▼
    Softmax → 概率分布
       │
       ▼
    Cross-Entropy Loss（与真实标签对比）
```

## 二、前向传播（Forward Propagation）详细流程

前向传播是**从输入到 Loss** 的完整计算路径。

### 流程图：

```
                         前向传播 (Forward Pass)
═══════════════════════════════════════════════════════════════════════

Step 0: 输入
   input_ids: [batch_size, seq_len]    例如 [4, 2048]（4个样本，每个2048个token）

   ┌──────────────────────────────────────────────────────────────────┐
   │ Step 1: Token Embedding                                          │
   │                                                                  │
   │   x = W_e[input_ids]          # 查表操作                         │
   │   W_e: [vocab_size, d_model]                                     │
   │   输出: [B, S, d_model]                                          │
   │                                                                  │
   │   FLOPs: 可忽略（查表，无矩阵乘法）                                │
   └──────────────────────────┬───────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step 2: Positional Encoding（以 RoPE 为例）                      │
   │                                                                  │
   │   x = x + pos_encoding      或  对 Q,K 施加旋转位置编码           │
   │                                                                  │
   │   FLOPs: O(B·S·d_model)，可忽略                                  │
   └──────────────────────────┬───────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │        ╔══════════════ Transformer Block #1 ══════════════╗      │
   │        ║                                                  ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3a: Pre-Attention LayerNorm             │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   μ = mean(x, dim=-1)                        │ ║      │
   │        ║  │   σ² = var(x, dim=-1)                        │ ║      │
   │        ║  │   x_norm = γ · (x - μ) / √(σ² + ε) + β      │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   γ, β: 可学习参数 [d_model]                  │ ║      │
   │        ║  └──────────────────┬──────────────────────────┘ ║      │
   │        ║                     │                             ║      │
   │        ║                     ▼                             ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3b: Multi-Head Attention                │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │ ① QKV 投影（每个 head 独立）：                │ ║      │
   │        ║  │    Q = x_norm @ W_q    [B,S,d] @ [d,d]       │ ║      │
   │        ║  │    K = x_norm @ W_k    [B,S,d] @ [d,d]       │ ║      │
   │        ║  │    V = x_norm @ W_v    [B,S,d] @ [d,d]       │ ║      │
   │        ║  │    然后拆分为多头: [B,S,h,d_head]             │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │ ② 缩放点积注意力（核心）：                     │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │    Score = Q @ K^T / √d_head                 │ ║      │
   │        ║  │           [B,h,S,S]                          │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │    ★ 应用 Causal Mask（下三角 mask）：        │ ║      │
   │        ║  │      Score[i,j] = -∞  当 j > i               │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │    Attn = Softmax(Score)                     │ ║      │
   │        ║  │    输出 = Attn @ V     [B,h,S,d_head]        │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │ ③ 拼接多头 + 输出投影：                       │ ║      │
   │        ║  │    concat = concat(head_0, ..., head_h-1)    │ ║      │
   │        ║  │    attn_out = concat @ W_o  [B,S,d]          │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │ FLOPs: QKV投影=2×3×B·S·d²                    │ ║      │
   │        ║  │        Q@K^T=2×B·h·S²·d_head                 │ ║      │
   │        ║  │        Attn@V=2×B·h·S²·d_head                │ ║      │
   │        ║  │        输出投影=2×B·S·d²                     │ ║      │
   │        ║  └──────────────────┬──────────────────────────┘ ║      │
   │        ║                     │                             ║      │
   │        ║                     ▼                             ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3c: 残差连接 + Dropout                  │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   x = Dropout(attn_out) + x   (残差连接)     │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   Dropout: mask ~ Bernoulli(1-p)             │ ║      │
   │        ║  │            output = input * mask / (1-p)      │ ║      │
   │        ║  │            (Inverted Dropout)                │ ║      │
   │        ║  └──────────────────┬──────────────────────────┘ ║      │
   │        ║                     │                             ║      │
   │        ║                     ▼                             ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3d: Pre-FFN LayerNorm                   │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   x_norm = LayerNorm(x)   （同 Step 3a）      │ ║      │
   │        ║  └──────────────────┬──────────────────────────┘ ║      │
   │        ║                     │                             ║      │
   │        ║                     ▼                             ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3e: Feed-Forward Network (FFN / MLP)    │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   ① 第一层线性变换 + 激活：                    │ ║      │
   │        ║  │      h = Activation(x_norm @ W₁ + b₁)        │ ║      │
   │        ║  │      W₁: [d_model, 4·d_model]               │ ║      │
   │        ║  │      激活函数: GELU / SiLU / ReLU            │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   ② 第二层线性变换：                          │ ║      │
   │        ║  │      ffn_out = h @ W₂ + b₂                  │ ║      │
   │        ║  │      W₂: [4·d_model, d_model]               │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   FLOPs: ①=2×B·S·d·4d = 8×B·S·d²           │ ║      │
   │        ║  │          ②=2×B·S·4d·d = 8×B·S·d²           │ ║      │
   │        ║  │          合计: 16×B·S·d²                     │ ║      │
   │        ║  └──────────────────┬──────────────────────────┘ ║      │
   │        ║                     │                             ║      │
   │        ║                     ▼                             ║      │
   │        ║  ┌─────────────────────────────────────────────┐ ║      │
   │        ║  │ Step 3f: 残差连接 + Dropout                  │ ║      │
   │        ║  │                                              │ ║      │
   │        ║  │   x = Dropout(ffn_out) + x                  │ ║      │
   │        ║  └─────────────────────────────────────────────┘ ║      │
   │        ║                                                  ║      │
   │        ╚══════════════════════════════════════════════════╝      │
   │                                                                  │
   │    ▲ 将输出 x 送入下一层（Block #2），重复 N 次                    │
   │    │                                                              │
   ▼    │  ┌──────────────────────────────────────────────────────────┐│
   │    └──│         Transformer Block #2 ... #N（同上）              ││
   │       └──────────────────────────────────────────────────────────┘│
   └──────────────────────────────────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step 4: Final LayerNorm                                          │
   │                                                                  │
   │   x = LayerNorm(x)                                               │
   └──────────────────────────┬───────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step 5: LM Head（输出投影）                                       │
   │                                                                  │
   │   logits = x @ W_out    # [B, S, d] @ [d, V] = [B, S, V]        │
   │   其中 V = vocab_size（词表大小，如 50000）                       │
   │                                                                  │
   │   FLOPs: 2×B·S·d·V                                               │
   └──────────────────────────┬───────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step 6: Loss 计算（训练时）                                       │
   │                                                                  │
   │   probs = Softmax(logits)       # [B, S, V]                      │
   │   loss = CrossEntropy(probs, target_ids)                         │
   │                                                                  │
   │   = -1/N · Σ log(probs[target_token])                            │
   │                                                                  │
   │   ★ 通常是"下一个 token 预测"：                                   │
   │     输入: "我 爱 北京"                                            │
   │     目标: "爱 北京 天安门"                                        │
   │     （每个位置预测下一个 token）                                   │
   └──────────────────────────────────────────────────────────────────┘
```

## 三、反向传播（Backpropagation）详细流程

反向传播是**从 Loss 到梯度**的计算路径，**完全沿着前向传播的逆序**逐层计算。

### 3.1 数学基础：链式法则

```
对于复合函数 L = f(g(h(x)))：

    ∂L/∂x = ∂L/∂f · ∂f/∂g · ∂g/∂h · ∂h/∂x

在计算图中，每个节点需要"上游梯度"和"本地梯度"：

    ∂L/∂input = ∂L/∂output · ∂output/∂input   ← 链式法则
                  ↑                ↑
              上游梯度         本地雅可比矩阵
```

### 3.2 实际实现中的关键

在实际框架（PyTorch/JAX）中，反向传播依赖 **计算图（Computation Graph）**：

```
前向传播时：
   - 每个操作（matmul, softmax, layernorm...）被记录为一个计算图节点
   - 每个节点的输入值被保存（用于反向计算梯度）
   - 这叫做 "保存中间激活"（activation checkpointing 可选择性丢弃以省显存）

反向传播时：
   - 从 Loss 节点出发，拓扑逆序遍历计算图
   - 每个节点根据"上游梯度"和"本地保存的前向值"计算"本地梯度"
   - 将梯度传递给输入节点
```

### 流程图：

```
                         反向传播 (Backward Pass)
═══════════════════════════════════════════════════════════════════════
                        （从 Loss 向上游逐层回溯）

   ┌──────────────────────────────────────────────────────────────────┐
   │ Step B1: Loss 反向                                               │
   │                                                                  │
   │   前向:  loss = CrossEntropy(Softmax(logits), target)             │
   │                                                                  │
   │   反向:  ∂loss/∂logits = probs - one_hot(target)                 │
   │          （Softmax + CrossEntropy 合并求导的经典结果）             │
   │          形状: [B, S, V]                                         │
   └──────────────────────────┬───────────────────────────────────────┘
                              │  dlogits: [B, S, V]
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step B2: LM Head 反向                                            │
   │                                                                  │
   │   前向:  logits = x @ W_out    [B,S,d] @ [d,V] = [B,S,V]         │
   │                                                                  │
   │   反向:                                                          │
   │     ∂loss/∂x     = dlogits @ W_out^T     [B,S,V] @ [V,d]         │
   │                    输出: [B, S, d]                                │
   │                                                                  │
   │     ∂loss/∂W_out = x^T @ dlogits       [B·S,d]^T @ [B·S,V]       │
   │                    输出: [d, V]                                   │
   └──────────────────────────┬───────────────────────────────────────┘
                              │  dx: [B, S, d]
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step B3: Final LayerNorm 反向                                     │
   │                                                                  │
   │   反向:                                                          │
   │     ∂loss/∂x  = LayerNorm_backward(dloss/dout, x_saved)          │
   │     ∂loss/∂γ  = Σ dloss/dout · (x-μ)/σ                          │
   │     ∂loss/∂β  = Σ dloss/dout                                     │
   └──────────────────────────┬───────────────────────────────────────┘
                              │  dx: [B, S, d]
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │        ╔══════════════ 逐层反向（从 Block #N 到 #1） ═══════════╗ │
   │        ║                                                        ║ │
   │        ║  ★ 每一层都接收到上游梯度 dL/d(out)，                      ║ │
   │        ║     计算对本层参数和输入的梯度，                          ║ │
   │        ║     然后传递给下一层（前向中的上一层）                     ║ │
   │        ║                                                        ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4f: 残差连接反向（第二残差）                 │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   前向:  out = ffn_out_do + x                     │  ║ │
   │        ║  │   反向:  d(ffn_out_do) = dout                     │  ║ │
   │        ║  │          d(x)          = dout   (加法直接传递)     │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ★ 残差连接的反向传播：梯度"分叉"，                  │  ║ │
   │        ║  │     一条流向 FFN，一条跳过 FFN 直接流向前面          │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   ║ │
   │        ║         ┌───────────┴──────────┐                        ║ │
   │        ║         ▼                      ▼                        ║ │
   │        ║  ┌──────────────┐    ┌──────────────────────┐           ║ │
   │        ║  │ d(ffn_do)    │    │ d(x) → 传给下一个步骤  │           ║ │
   │        ║  └──────┬───────┘    └──────────────────────┘           ║ │
   │        ║         │                                               ║ │
   │        ║         ▼                                               ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4e: Dropout 反向（第二 Dropout）            │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   前向:  out = input * mask / (1-p)               │  ║ │
   │        ║  │   反向:  d(input) = dout * mask / (1-p)           │  ║ │
   │        ║  │          （Trick → 0 的时候梯度也被置零）          │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   ║ │
   │        ║                     ▼                                   ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4d: FFN 反向                                │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   前向:                                           │  ║ │
   │        ║  │     h   = Activation(x_norm @ W₁ + b₁)            │  ║ │
   │        ║  │     out = h @ W₂ + b₂                            │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   反向（从后往前）：                                │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ① ∂L/∂h   = dout @ W₂^T     [B,S,d] @ [d,4d]   │  ║ │
   │        ║  │      ∂L/∂W₂ = h^T @ dout      [B,S,4d]^T @ ...   │  ║ │

   │        ║  │   ③ ∂L/∂(x_norm) = dh_pre_act @ W₁^T              │  ║ │

   │        ║  │      ∂L/∂b₁       = Σ dh_pre_act                  │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   ║ │
   │        ║                     ▼                                   ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4c: Pre-FFN LayerNorm 反向                 │  ║ │
   │        ║  │   同 Step B3                                      │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   ║ │
   │        ║         ┌───────────┴──────────┐                        ║ │
   │        ║         ▼                      ▼                        ║ │
   │        ║  ┌──────────────┐    ┌──────────────────────┐           ║ │
   │        ║  │ d(x) → 准备   │    │ 与残差分支梯度汇合    │           ║ │
   │        ║  │ 进入残差汇合   │    │                      │           ║ │
   │        ║  └──────┬───────┘    └──────────┬───────────┘           ║ │
   │        ║         │                       │                       ║ │
   │        ║         └───────────┬───────────┘                       ║ │
   │        ║                     ▼                                   ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4b: 残差连接反向（第一残差 + Dropout）      │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   d(attn_do) = dout   (残差分支)                   │  ║ │
   │        ║  │   d(x_skip)  = dout   (跳过注意力，直连分支)       │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   再经过第一 Dropout 反向: d(attn_out)=d(attn_do)  │  ║ │
   │        ║  │                                   * mask/(1-p)    │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   │ ║ │
   │        ║                     ▼                                   │ ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4a: Multi-Head Attention 反向（核心）       │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ① 输出投影反向：                                 │  ║ │
   │        ║  │      d(concat) = d(attn_out) @ W_o^T              │  ║ │
   │        ║  │      dW_o      = concat^T @ d(attn_out)           │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ② 拆分回多头: d(concat) → [d_head0, d_head1,...]│  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ③ 注意力核心反向（最关键！）：                     │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │      前向:                                         │  ║ │
   │        ║  │        S = Q @ K^T / √d                           │  ║ │
   │        ║  │        A = Softmax(S)                             │  ║ │
   │        ║  │        O = A @ V                                  │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │      反向:                                         │  ║ │
   │        ║  │        dV = A^T @ dO              [B,h,S,S]^T     │  ║ │
   │        ║  │        dA = dO @ V^T              [B,h,S,dh]      │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │        ★ Softmax 反向（雅可比是 diag(p)-pp^T）：   │  ║ │
   │        ║  │        dS = A · (dA - Σ(A·dA, dim=-1))           │  ║ │
   │        ║  │        等价于: dS = A ⊙ dA - A·Σ(A⊙dA)           │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │        ★ 除以 √d 的反向:                           │  ║ │
   │        ║  │        dS_scaled = dS / √d                        │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │        dQ = dS_scaled @ K                        │  ║ │
   │        ║  │        dK = dS_scaled^T @ Q   (或 dS_scaled @ Q)  │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │   ④ QKV 投影矩阵的梯度：                           │  ║ │
   │        ║  │      dW_q = x_norm^T @ dQ                        │  ║ │
   │        ║  │      dW_k = x_norm^T @ dK                        │  ║ │
   │        ║  │      dW_v = x_norm^T @ dV                        │  ║ │
   │        ║  │                                                   │  ║ │
   │        ║  │      d(x_norm) = dQ@W_q^T+dK@W_k^T+dV@W_v^T      │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   │  ║ │
   │        ║                     ▼                                   │  ║ │
   │        ║  ┌──────────────────────────────────────────────────┐  ║ │
   │        ║  │ Step B4pre: Pre-Attention LayerNorm 反向         │  ║ │
   │        ║  └──────────────────┬───────────────────────────────┘  ║ │
   │        ║                     │                                   │  ║ │
   │        ║         ┌───────────┴──────────┐                        │  ║ │
   │        ║         ▼                      ▼                        │  ║ │
   │        ║  ┌──────────────┐    ┌──────────────────────┐           │  ║ │
   │        ║  │ d(x_norm)    │    │ d(x_skip)            │           │  ║ │
   │        ║  └──────┬───────┘    └──────────┬───────────┘           │  ║ │
   │        ║         │                       │                       │  ║ │
   │        ║         └───────────┬───────────┘                       │  ║ │
   │        ║                     ▼                                   │  ║ │
   │        ║         ┌────────────────────────┐                      │  ║ │
   │        ║         │  汇合: d(x) = 两路梯度之和 │                   │  ║ │
   │        ║         └────────────┬───────────┘                      │  ║ │
   │        ║                      │                                  │  ║ │
   │        ╚══════════════════════╧══════════════════════════════════╝ │
   │                                │                                   │
   │              ┌─────────────────┘                                   │
   │              ▼                                                     │
   │    将 d(x) 作为上游梯度                                           │
   │    传递给前一层（Block #N-1）                                      │
   │    ...                                                            │
   │    重复直到 Block #1                                               │
   └──────────────────────────────────────────────────────────────────┘
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │ Step B5: Token Embedding 反向                                     │
   │                                                                  │
   │   前向:  x = W_e[input_ids]   (查表/索引操作)                     │
   │                                                                  │
   │   反向:                                                          │
   │     ∂L/∂W_e = scatter_add(dx, input_ids)                         │
   │     （将梯度按 token id 累加到对应的 embedding 行）               │
   │                                                                  │
   │     d(input_ids) 不存在（离散值，不可导）                         │
   └──────────────────────────────────────────────────────────────────┘

最终结果：
   ★ 每个参数矩阵/向量都得到了梯度：dW_q, dW_k, dW_v, dW_o, dW₁, dW₂, dγ, dβ, dW_e, dW_out
   ★ 优化器使用这些梯度来更新参数（AdamW / SGD 等）
```

## 四、反向传播中的激活值显存占用

反向传播需要前向传播保存的**中间激活**，这在 LLM 训练中是最大的显存开销之一：

```
┌────────────────────────────────────────────────────────────────┐
│              前向时保存的激活（用于反向计算）                  │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  每个 Transformer Block 需要保存：                              │
│                                                                │
│  ① Q, K, V（注意力计算需要）      → B·S·d_model × 3           │
│  ② Softmax 输入（Softmax 反向需要） → B·h·S·S                 │
│  ③ Attention 输出                  → B·S·d_model              │
│  ④ LayerNorm 输入（算 μ, σ 用）    → B·S·d_model × 2          │
│  ⑤ FFN 第一层激活值                → B·S·4d_model             │
│  ⑥ Dropout mask                    → B·S·d_model × 2          │
│                                                                │
│  总计（粗略）：~ B·S·d_model·(10 + 4·h·S/d_model)             │
│                                                                │
│  例如 GPT-3 (175B): d=12288, S=2048, B=1                       │
│  每层激活 ≈ 2048×12288×(10+...) ≈ 250MB/层 × 96层 ≈ 24GB      │
│  加上模型权重 ≈ 350GB (FP16)                                   │
│  总需求远超单 GPU 显存 → 需要模型并行 + 激活重计算              │
└────────────────────────────────────────────────────────────────┘
```

## 五、关键要点总结

| 维度 | 前向传播 | 反向传播 |
|------|---------|---------|
| **方向** | 输入 → 输出（自底向上） | Loss → 参数（自顶向下） |
| **依赖** | 逐层计算，每层依赖上一层输出 | 链式法则，需要上游梯度 + 前向保存的激活值 |
| **计算量** | ~2P FLOPs/token（P=参数量） | ~4P FLOPs/token（约前向的 2 倍） |
| **显存** | 存模型权重 + KV Cache（推理）/ 激活值（训练） | 存梯度 + 优化器状态（训练时显存 ≈ 前向的 3~4 倍） |
| **残差连接** | 加法短路，缓解梯度消失 | 梯度分叉，一条流过变换、一条直通，保证梯度流通 |
| **LayerNorm** | 归一化稳定训练 | 可学习参数 γ, β 有梯度 |
| **Softmax** | 稳定实现用 log-sum-exp trick | 雅可比矩阵 diag(p) - ppᵀ |
| **Dropout** | 训练时 mask 并放大 1/(1-p)，推理时关闭 | 梯度和 mask 做相同缩放 |
